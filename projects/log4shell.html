Log4Shell Exploit Analysis

Security Research Completed
Java Log4j Exploit Research Mitigation

Project Overview

Recreated the CVE-2021-44228 "Log4Shell" vulnerability in a controlled environment to observe the full remote code execution chain. The study examined how JNDI lookups in Log4j enable attackers to trigger outbound connections and run arbitrary classes.

Environment & Tools

  • Vulnerable Java application using Log4j 2.14.1
  • Marshalsec LDAP server and Python HTTP server for payload delivery
  • curl and custom scripts to inject ${jndi:} payloads
  • Wireshark and log monitoring to validate exploit execution

Key Experiments

  • Demonstrated end-to-end exploitation from log injection to remote class loading.
  • Crafted obfuscated payloads using lookup functions and encoding to bypass simple filters.
  • Evaluated mitigations such as patched Log4j versions, formatMsgNoLookups, and network egress controls.

Lessons Learned

  • Logging features can become critical attack surfaces when misconfigured.
  • Effective detection must normalize Log4j lookups to defeat evasion tricks.
  • Layered defenses—patching, configuration hardening, and outbound filtering—provide the strongest protection.
